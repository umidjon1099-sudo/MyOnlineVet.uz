Пошаговый запуск Telegram Mini/Web App (простыми словами)

1) Подготовьте сервер и домен
- Нужен сервер (VPS/хостинг), где будет работать ваш бот.
- Нужен домен (например: mypetbot.uz).
- Домен должен открываться по HTTPS (SSL обязателен).
  Telegram Web App не открывается с обычного http.

2) Разместите проект на сервере
- Скопируйте ваш проект на сервер.
- Установите зависимости из requirements.txt.
- Проверьте, что бот запускается командой:
  python main.py

3) Заполните переменные окружения
Создайте .env (или настройте переменные в панели хостинга) и укажите:

- BOT_TOKEN=ваш_токен_бота
- WEB_APP_URL=https://ваш-домен/webapp
- WEBAPP_HOST=0.0.0.0
- WEBAPP_PORT=8080

Важно:
- WEB_APP_URL должен быть точно с /webapp в конце.
- Пример: https://mypetbot.uz/webapp

4) Настройте прокси (Nginx), чтобы домен вел на порт Web App
Логика должна быть такой:
- https://ваш-домен/webapp -> проксируется на http://127.0.0.1:8080/webapp
- https://ваш-домен/webapp/api/... -> на тот же backend

То есть внешний HTTPS, внутри можно локальный http до python процесса.

5) Настройте домен в BotFather
Откройте BotFather и сделайте:
- /mybots
- выберите вашего бота
- Bot Settings -> Menu Button (или Web App)
- укажите URL: https://ваш-домен/webapp

Также при необходимости:
- /setdomain
- укажите ваш домен (без пути), например: mypetbot.uz

6) Перезапустите бота
После изменения env и прокси перезапустите сервис бота:
- systemctl restart ... (если через systemd)
или
- docker compose up -d --build (если через Docker)
или просто
- python main.py

7) Проверка в Telegram
- Откройте вашего бота.
- Нажмите кнопку Mini App в меню.
- Должно открыться встроенное окно Web App.

8) Если не открывается, проверьте по чек-листу
- URL точно HTTPS?
- URL точно совпадает с WEB_APP_URL?
- В BotFather указан тот же домен/URL?
- Порт 8080 доступен локально приложению?
- Прокси правильно прокидывает /webapp и /webapp/api?
- Бот реально запущен и без ошибок?

9) Для локальной разработки (без Telegram)
Можно временно включить dev-режим:
- WEBAPP_DEV_USER_ID=123456789
Это позволяет открывать /webapp в браузере без Telegram initData.
Для production лучше оставить пустым.

Коротко: что обязательно для продакшена
- Рабочий HTTPS домен
- WEB_APP_URL=https://домен/webapp
- Настройка URL в BotFather
- Запущенный бот и webapp_server (он стартует вместе с main.py)
